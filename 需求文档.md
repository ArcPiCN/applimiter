
# Vibe Coding：短视频依赖管理助手

## 产品概述

一款安卓应用，通过无障碍服务帮助用户管理短视频使用时间。用户可以精确设置每个 App 内特定页面的允许停留时长，超时后自动强制退出，帮助戒断短视频依赖。

---

## 核心功能

### 1. 无障碍服务监听
- 实时监听用户打开的 App 和页面（通过包名 + Activity 名识别）
- 根据用户预设的规则，在目标页面启动倒计时
- 倒计时归零时执行全局返回操作，强制退出目标页面
- 用户切换到其他页面或退出目标 App 时，自动取消/重置计时器

### 2. 规则配置界面（主界面）
用户可以：
- 从已安装 App 列表中选择目标应用
- 为该 App 的特定页面设置允许停留时长
- 自定义添加页面规则（通过运行时捕获当前 Activity 名）
- 启用/禁用/编辑/删除已有规则
- 为每个规则自定义页面别名（方便识别，如“视频详情页”、“视频号”）

### 3. 倒计时提示
- 在目标页面停留期间，通过 Toast 显示剩余时间
- 最后 10 秒时通过系统通知提醒用户

### 4. 预置模板
- 预置抖音、微信等常见 App 的已知 Activity 名映射
- 用户添加规则时可直接选择预设页面，无需手动捕获

### 3. `AppDatabase.kt` - Room 数据库

```kotlin
@Database(entities = [RuleEntity::class], version = 1, exportSchema = false)
abstract class AppDatabase : RoomDatabase() {
    abstract fun ruleDao(): RuleDao

    companion object {
        @Volatile
        private var INSTANCE: AppDatabase? = null

        fun getInstance(context: Context): AppDatabase {
            return INSTANCE ?: synchronized(this) {
                val instance = Room.databaseBuilder(
                    context.applicationContext,
                    AppDatabase::class.java,
                    "app_limiter_db"
                ).build()
                INSTANCE = instance
                instance
            }
        }
    }
}
```

### 7. `PageLimitService.kt` - 无障碍服务（核心）

这是最核心的文件，需要实现以下逻辑：

```
State 管理：
- currentPackage: 当前前台 App 的包名
- currentActivity: 当前前台 Activity 名
- currentTimerJob: 当前正在运行的计时协程
- toastJob: 显示 Toast 的协程

事件处理（onAccessibilityEvent）：
1. 只处理 TYPE_WINDOW_STATE_CHANGED 事件
2. 获取 event.packageName 和 event.className
3. 如果包名变了，取消所有计时器
4. 从数据库查询当前页面的规则（按包名+Activity名匹配）
5. 如果匹配到启用状态且允许时长>0的规则：
   - 启动协程计时器，delay(允许秒数) 后执行 performGlobalAction(GLOBAL_ACTION_BACK)
   - 同时启动一个每5秒触发一次的 Toast 提示协程，显示剩余时间
   - 如果剩余时间<=10秒，发送系统通知提醒
6. 如果切换到同一个 App 的不同页面：
   - 取消旧计时器
   - 按新页面的规则重新计时
7. 如果切换到不同 App：
   - 取消所有计时器

生命周期：
- onServiceConnected: 初始化，创建通知渠道，以前台服务方式运行
- onDestroy: 取消所有协程
- onUnbind: 清理资源
```

关键实现细节：
- 使用 `CoroutineScope(Dispatchers.Main)` 管理计时器
- 计时 Job 需保存引用，用于取消
- 前台服务通知需包含"正在运行"的文字
- `performGlobalAction(GLOBAL_ACTION_BACK)` 在协程中调用

### 8. MainViewModel.kt - 主界面逻辑

```kotlin
class MainViewModel(application: Application) : AndroidViewModel(application) {
    private val repository = (application as AppLimiterApplication).repository
    val allRules: StateFlow<List<RuleEntity>> = repository.allRules
        .stateIn(viewModelScope, SharingStarted.Eagerly, emptyList())

    fun deleteRule(rule: RuleEntity) {
        viewModelScope.launch { repository.delete(rule) }
    }

    fun toggleRule(id: Long, enabled: Boolean) {
        viewModelScope.launch { repository.setEnabled(id, enabled) }
    }

    fun addRule(
        packageName: String,
        appName: String,
        activityName: String,
        pageLabel: String,
        maxStaySeconds: Int
    ) {
        viewModelScope.launch {
            repository.insert(
                RuleEntity(
                    packageName = packageName,
                    appName = appName,
                    activityName = activityName,
                    pageLabel = pageLabel,
                    maxStaySeconds = maxStaySeconds
                )
            )
        }
    }
}
```
